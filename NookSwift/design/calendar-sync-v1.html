需要你确认的几个细节

1. ICS 格式：用 VEVENT（全天事件）还是 VTODO（reminder 风格）？
    → 我推荐 VEVENT，更通用、跨日历软件兼容（Outlook/Google Calendar 都直观）

2. 同步方向：v1 单向 Nook → Reminders（Reminders 修改不回流）
    → 反向同步代码量大且容易出冲突，先单向稳定运行

3. Reminders 列表名：固定叫「Nook」还是允许用户改？
    → 我推荐固定，避免用户改了之后找不到

4. 同步触发时机：每次任务变更（add/done/rm/edit）实时触发，debounce 500ms
    → 还是手动按钮触发？我推荐自动 + 设置里有"重新全量同步"兜底

5. 已完成任务：在 Reminders 里也标完成（保留可见）还是删除？
    → 我推荐"标完成"，跟 Reminders 用户习惯一致

6. 没 dueDate 的任务：要不要也同步进 Reminders（无截止日期的提醒）？
    → 我倾向**只同步有 dueDate 的**，避免 Reminders 列表被 Nook 杂项淹没
