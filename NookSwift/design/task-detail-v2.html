Task Detail UI · 3 个优化方案

对比 New Task / Edit Task 详情页的 3 个布局方向。聚焦在 Add subtask 输入区的视觉权重 + 整体节奏。 当前实现的问题:add subtask 行像静态文本不像输入框、4 条分隔线把页面切碎、subtasks 列表跟"添加"行视觉割裂。

方案 A · 紧凑现代式 ⭐ 推荐

核心思路: 元数据用 inline chip 紧凑排,无分隔线;subtasks 整体用一个浅灰底色的"组",列表 + 输入框在同一组内视觉连续,添加 subtask 一眼看出是输入区。
视觉节奏: 标题区(白) → 元数据 chips → 标签 chips → subtasks 组(浅灰底) → 留白。无横线,呼吸感来自背景对比 + 间距。

方案 B · 卡片分组式

核心思路: 每个分组(meta / tags / subtasks)用浅灰圆角卡片包起来,卡内部紧凑,卡之间留白。标题保持白底无 chrome,展示主体内容。
视觉节奏: 标题区(白) → 3 张紧凑卡片(浅灰)。比 A 更清晰分组,但视觉块更多、稍重。

方案 C · Notion / Linear 风(极简)

核心思路: 标题超大,元数据用一行 inline 标签 + 分隔点(像 Notion document properties),底部一条线分隔。subtasks 不用任何 chrome,直接列出来,Add 行的 + 用虚线 box,跟列表 checkbox 视觉权重一致,节奏统一。
视觉节奏: 标题超大(22pt) → 元数据 inline 一行 → 分隔线 → subtasks(无 chrome)。最纯粹,但分组弱,数据多时可能略密。

三方案 trade-off 总结

A · 紧凑现代 B · 卡片分组 C · Notion/Linear
Add subtask 一眼看出是输入框 ✅ 浅灰组里的虚线分隔 + 输入 ✅ 卡片内最后一行,白底框 ⚠️ 虚线 box + 输入,稍弱
视觉块数 2 个(标题区 + subs 组) 4 个(标题区 + 3 卡) 1 整块(无明显分组)
空态简洁度 高(空 chip + 简单输入) 中(3 张半空卡) 最高(纯净)
数据多时层次 清晰 最清晰 略密
改 Swift 工作量 中(一个新组件) 大(3 个卡片重构) 小(去掉框,改样式)
参照 Things 3 + Linear 折中 Bear / Apple 备忘录 Notion / Linear

我的推荐: 方案 A。理由:解决你说的"add subtask 不像输入框"问题最直接(浅灰底色组里 visually 一体);视觉块少,符合英语 Mac 用户偏好的 minimal;改造工作量适中。
