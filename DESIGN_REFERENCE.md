# 滴答清单（TickTick/Dida365）Light Mode UI 设计规范

基于对 dida365.com 官网 CSS 源码、Web App 结构、第三方自定义样式表（VidFerris/TickTickStylesheet）及产品分析文档的综合研究，整理如下设计规范。

## 一、色彩体系（Color Palette）

### 1.1 品牌主色

| 用途 | 色值 | 说明 |
|------|------|------|
| 品牌蓝/主色调 | `#4772FA` | 全局强调色，按钮、链接高亮、选中状态 |
| 主色悬停态 | `#3E60CD` | 按钮 hover 加深 |
| 主色浅底 | `rgba(71,114,250,0.1)` | 选中项背景、轻量高亮 |
| 主色半透明 | `rgba(71,114,250,0.8)` | 渐变/遮罩 |

### 1.2 优先级颜色

| 优先级 | 色值（原生默认） | 自定义样式参考 | 旗帜标记 |
|--------|-----------------|---------------|---------|
| 高优先级（High） | `#EB4D3D`（红色系） | `#FF3180` | 红旗 |
| 中优先级（Medium） | `#FF9A14`（橙黄色系） | `#FFC817` | 橙旗 |
| 低优先级（Low） | `#4B7BEC`（蓝色系） | `#4B6FDE` | 蓝旗 |
| 无优先级（None） | `#B8B8B8`（灰色） | `#AAAAAA` | 无旗帜 |

> 说明：原生默认色值基于产品文档和行业对比推断，自定义样式参考值来自 VidFerris 的开源 CSS 中对 `.priority-high`、`.priority-medium`、`.priority-low` 类名的覆写。实际 Web App 中红/橙/蓝/灰为固定的优先级颜色。

## 二、字体排版（Typography）

### 2.1 字体族

```css
font-family: system-ui, -apple-system, BlinkMacSystemFont, 
             "Segoe UI", Roboto, Helvetica, Arial, sans-serif,
             "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol";
```

> 中文环境下会回退到系统默认的 PingFang SC（macOS）/ Microsoft YaHei（Windows）。这是标准的系统字体栈，确保各平台最佳渲染效果。

### 2.2 字号体系

| 用途 | 字号 | 字重 |
|------|------|------|
| 页面大标题 | 36px ~ 48px | 600 ~ 700 |
| 区块标题 | 24px ~ 28px | 600 |
| 子标题 | 18px ~ 20px | 600 |
| 导航项 | 16px | 500 |
| 任务标题 | 14px ~ 16px | 400 |
| 任务描述/次要文字 | 13px ~ 14px | 400 |
| 标签/小文字 | 12px | 400 |
| 极小标注 | 10px | 400 |

### 2.3 行高

- 任务项行高：`20px`（紧凑模式参考）
- 导航行高：`80px`（桌面端头部）/ `44px`（移动端头部）
- 菜单项行高：`32px` ~ `34px`
- 正文段落行高：`1.25` ~ `1.5`

## 三、任务项设计（Task Item）

### 3.1 复选框样式

- **形状**：圆形（`border-radius: 50%`）
- **图标类名**：`.icon-checkbox`
- **默认颜色**：灰色边框（`#B8B8B8` ~ `#AAAAAA`）
- **优先级着色**：复选框颜色根据优先级变化
  - 高优先级：红色圆圈（`#EB4D3D`）
  - 中优先级：橙黄色圆圈（`#FF9A14`）
  - 低优先级：蓝色圆圈（`#4B7BEC`）
  - 无优先级：灰色圆圈
- **完成状态**：勾选后显示对勾（checkmark），圆圈填充对应优先级颜色
- **子任务复选框**：使用 `.icon-real-subtask` / `.icon-subtask` 类名，同样支持优先级着色

### 3.2 任务项布局

```
[圆形复选框] [任务标题文字] [日期/标签/附件图标]  [...]菜单
     ↑               ↑                    ↑               ↑
  左侧对齐      中间弹性填充        右侧辅助信息     操作菜单
```

- **CSS 类名结构**：`.task > .t-inner > .t-check + .title-wrap + .tips`
- **内边距**：左右 `6px` ~ `12px`
- **任务背景区域**：`.l-task-bg`，`margin: 0 16px`，`border-radius: 4px`
- **任务高度**：默认约 `36px` ~ `40px`（紧凑模式 `20px`）
- **悬停效果**：背景变为 `rgba(25,25,25,0.03)`

### 3.3 已完成任务

- 任务标题添加**删除线**（strikethrough）
- 文字颜色变淡（降低不透明度）
- 复选框显示对勾
- 已完成任务可折叠收起，默认折叠在"已完成"分组下

### 3.4 子任务

- 左侧缩进约 `24px`（2 字符宽度）
- 使用 `.icon-real-subtask` 图标
- 缩进的子任务列表可展开/折叠

## 四、输入框设计（Add Task Input）

### 4.1 位置与样式

- **位置**：任务列表顶部（`.tl-quick`）
- **CSS 类名**：`.tl-quick .task-input`
- **内边距**：左右 `6px` ~ `12px`
- **外边距**：`margin: 0 28px 20px`（标准）/ `margin: 0 20px 2px`（紧凑）
- **提示文字**："添加任务" / "Add Task"
- **交互**：点击后展开，显示日期选择器、优先级选择器、清单选择等快捷工具栏

### 4.2 快速添加栏工具按钮

- 日期设置：`.set-duedate`
- 优先级设置：`.set-priority`
- 清单选择
- 标签管理
- 附件按钮

## 五、侧边栏/清单导航设计（Sidebar）

### 5.1 布局结构

三栏布局（Three-Panel Layout）：

```
┌─────────┬──────────────────┬─────────────┐
│ 侧边栏   │    任务列表       │  任务详情    │
│ .g-left  │   .g-center      │  .g-right   │
│ 200~260px│    弹性宽度       │  可折叠面板  │
└─────────┴──────────────────┴─────────────┘
```

- **侧边栏宽度**：默认 `260px`（可调为 `200px`）
- **容器 ID**：`#project-list-view`，滚动区域 `#project-list-scroller`
- **中间面板**：`#task-list-view`，`.g-center`
- **右侧详情面板**：`.g-right`，任务详情弹窗

### 5.2 侧边栏内容

- 顶部：用户头像（48px 圆形）
- 智能清单区域：
  - 收集箱 / Inbox
  - 今天 / Today
  - 未来7天 / Next 7 Days
  - 日历 / Calendar
- 清单列表区域（`.lists`）：
  - 清单名称 + 颜色标识条（左侧彩色竖线）
  - 未完成任务数量徽标
  - 文件夹（可展开/折叠）
- 底部：
  - 升级按钮（`.upgrade-button`）
  - 标签管理
  - 设置入口

### 5.3 清单颜色标识

每个清单可设置独立颜色，该颜色会：
- 在侧边栏清单名称左侧显示为竖线/圆点
- 在任务列表中作为任务左侧的彩色标记条
- 在日历视图中作为任务背景色

## 十三、设计总结

滴答清单的设计风格可以总结为：

1. **简洁克制**：大面积留白，元素间距舒适，视觉层次清晰
2. **系统字体优先**：不使用自定义字体，依赖系统字体栈确保跨平台一致性
3. **品牌蓝 `#4772FA`** 作为唯一强调色贯穿始终
4. **圆形复选框**配合优先级颜色，是核心视觉识别元素
5. **三栏布局**（侧边栏 + 任务列表 + 详情面板）为经典的 GTD 工具布局
6. **低饱和度的灰色体系**用于背景和文字，减少视觉噪音
7. **圆角偏大**（12px ~ 16px），营造柔和现代感
8. **阴影使用克制**，仅在浮层和卡片上使用轻柔阴影
